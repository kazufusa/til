# compare-table スキル

ユーザが複数の項目(ツール、ライブラリ、サービスなど)の比較を求めている時に使う。

## 出力手順

1. コンテキストから比較対象を抽出する。
2. 最初の block を Markdown の表にする。列は必ず `| 名前 | 特徴 | 出典 |`。
   - 「特徴」はコンテキストの記述に基づく1〜2文。
   - 「出典」列にはそのデータ元になった chunk のファイル名を書く。
3. 表の各行の根拠 chunk_id を、その block の citations に漏れなく入れる。
4. 表のあとに、選び方の短い補足 block を1つ付ける(これにも citations を付ける)。

## 禁止

- コンテキストに無い項目を表に足さない。
- 「特徴」を一般論で埋めない。根拠が無い行は作らない。
