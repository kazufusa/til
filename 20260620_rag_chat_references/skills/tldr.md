# tldr スキル

ユーザが「要するに」「ざっくり」「まとめて」など、短い要約を求めている時に使う。

## 出力手順

1. 回答は最大でも blocks を 5 個までにする。
2. 各 block は1文の箇条書き(先頭に "- ")にする。
3. 各 block には、その要点の根拠 chunk_id を citations に必ず入れる。
4. 細かい列挙や具体例は省き、結論・要点だけを残す。

## 禁止

- 6個以上に増やさない。
- 根拠 chunk の無い要点を書かない。
