# マルチAIエージェントによる実在部署の仮想組織化 PoC 報告書（作成中）

このファイルは、最終成果物であるWord報告書の下敷き（骨格）。
中身はWord（同時編集）に移して育てる。この目次と数字テンプレをそのまま使う。
運用ルール:

- 清書はしない。見出しの下は箇条書きで足していく
- 細かい技術内容はGitHub（リポジトリのdocsやコード）に書き、Wordからリンクする
- 担当は「編集」と「技術オーナー」の2段で持つ。編集＝貢献を束ねて書く人、技術オーナー＝中身を設計・保証する人。テックリードが編集を持つ節でも、技術オーナーは別名にする（編集と設計を1語に潰さない）
- 各節は担当が更新する。空欄の節が未達の可視化になる
- 数字は各節共通の型（下記テンプレ）で、下の記録ルールに従って載せる。揃っていない数字は比較表に載せない
- 決定の正本はGitHubの DECISIONS.md。下の決定ログは清書時に転記する要約

各節に載せる数字の共通テンプレ:

| 項目 | 値 |
| --- | --- |
| PoC / タスク名 | |
| 使用モデル | |
| 実行日 | |
| 精度（測定法と値） | |
| トークン量（入力 / 出力 / 合計） | |
| 概算コスト | |
| 備考 | |

数字の記録ルール（比較を成立させるための最小限。テックリードが揃える）:

- トークン: システムプロンプトとリトライ分を含める。マルチエージェントはPoC単位で合算。APIのusageを正とする
- モデル・実行日: 必ず記録する（モデルが違えば精度もコストも変わる）
- コスト: モデル別の単価表から算出（入力×入力単価＋出力×出力単価）

精度の測り方（何を正解とするか）は評価の設計そのもので、正解定義オーナーが4章・5章で定める。ここでは扱わない。
